The agent

What I built, and where a human was needed

A five-stage pipeline: an LLM researches into a strict schema, an adversarial agent re-checks a sample, then Python clusters and renders. Composio's SDK/MCP slots in at stage ①.

What the agent does well

  • Fan-out at scale: 100 apps into one comparable schema, evidence URL required per row; no free text to parse.
  • Grounded, not guessed: it opens the official auth + API pages and confirms MCP from the vendor, not a blog.
  • Adversarial self-check: a second agent is told to disprove the first pass, so agreement is real signal.

Where a human was needed (honest)

  1. self-serve vs mixed: "free sandbox, paid/approved production" (Plaid, Devin, WhatsApp). The agent over-rounds to "gated"; a human made the call.
  2. official vs community MCP: the agent under-counts brand-new first-party MCPs; a human confirmed Plain / Plaid / Amazon.
  3. long-tail unknowns: fanbasis, Paygent, iPayX, Waterfall, Consensus have thin docs; "blocked / needs human check" is the honest output, not an invented answer.
  4. stale facts: a human caught Amazon SP-API dropping AWS SigV4 (~2023).

      Reproduce it

      Run the pipeline

      Every number on this page comes from the committed dataset; patterns.py needs no API key.

      # research all 100 into the strict schema (live web search)
python src/research_agent.py            # → data/results.raw.json
# independent adversarial re-check on the stratified sample
python src/verify_agent.py              # → data/verification.json
# cluster + print every distribution on this page (no API key needed)
python src/patterns.py                  # → data/patterns.json
# rebuild this page from the three JSON files
python src/build_site.py               # → site/index.html
      The takeaway

      Build where it's a green field, not where it's gated.
